ORM - механизм,  который позволяет отобразить сущности бд в объекты используемого языка программирования.
* база данных - объект contextDB
* таблица - коллекция объектов
* строка таблицы - объект
* структура таблицы - класс

для работы sequelize с MSSQL необходим пакет tedious

sequelize-схема - описание структуры таблиц бд
описать структура таблицы можно:
* с помощью метода define(nameModel, struct)
* наследоваться от класса Model и вызвать статический метод init с параметром описывающим структуру

select - findAll(options)
insert - create(obj)
update - update(options)
delete - destroy(options)

Хуки (также известные как события жизненного цикла) — это функции, которые вызываются до и после выполнения вызовов в sequelize.

hasMany - указывает на отношение между таблицами(один ко многим)
A.hasMany(B, options) 
A.hasMany(B, {foreignKey: 'fk_B'})

raw query - позволяет обратиться к бд через строку sql-запррос.

code first - подход, описываем модели на основании которых будут созданы таблицы бд.

транзакция - набор операций в которой либо выполнятся все, либо не выполнится ни одна
